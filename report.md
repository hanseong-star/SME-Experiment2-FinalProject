# DQN 기반 RTT 위치 추정 실험 보고서

## 1. 모티베이션 & 인트로

본 프로젝트의 목표는 18개 기지국에서 측정된 RTT 기반 거리값을 이용하여 사용자의 2차원 위치를 추정하는 것이다. 입력 데이터는 각 사용자에 대해 18개의 거리 측정값을 제공하며, 출력은 사용자 위치 \((x, y)\)이다. 데이터의 실제 좌표 범위는 대략 \(x=-60\sim60\), \(y=-30\sim30\)이므로 전체 공간은 약 \(120m \times 60m\) 크기의 맵으로 볼 수 있다.

일반적인 위치 추정 문제에서는 삼변측량이나 least squares 기반 방법을 먼저 떠올릴 수 있다. 그러나 RTT 거리값에는 평균적인 bias뿐 아니라 샘플마다 크게 흔들리는 오차가 포함되어 있다. 따라서 단순한 기하학적 계산만으로는 모든 샘플에서 안정적인 성능을 얻기 어렵다. 이러한 문제를 다른 관점에서 보기 위해, 본 실험에서는 위치 추정을 한 번에 회귀하는 방식이 아니라 에이전트가 맵 위를 이동하면서 정답 위치 근처에서 멈추도록 학습하는 강화학습 문제로 재정의하였다.

비교 기준으로는 물리 기반 RTT 보정 + Weighted Robust Least Squares + Residual Correction 실험을 사용하였다. 이 비교 실험은 기지국별 bias와 신뢰도를 명시적으로 계산하고, Ridge Regression으로 잔여 오차를 보정하는 방식이다. 반면 본 보고서의 중심은 DQN 기반 강화학습 모델이며, 물리 기반 실험은 강화학습 결과가 어느 정도 수준인지 판단하기 위한 비교용 baseline으로만 사용하였다.

전체 실험의 핵심 질문은 다음과 같다.

| 질문 | 의미 |
|---|---|
| RTT 위치 추정 문제를 강화학습 환경으로 바꿀 수 있는가? | 위치 추정을 순차적 의사결정 문제로 모델링할 수 있는지 확인한다. |
| DQN이 이동과 정지 행동을 통해 위치를 찾을 수 있는가? | 거리값 18개와 현재 위치를 상태로 사용했을 때 정책 학습이 가능한지 확인한다. |
| 보상 설계가 성능에 어떤 영향을 주는가? | STOP 보상, 이동 보상, 성공 반경 변화에 따른 성능 변화를 분석한다. |
| 물리 기반 baseline과 비교했을 때 어떤 한계가 있는가? | 강화학습 접근의 장점과 단점을 공정하게 해석한다. |

## 2. 알고리즘 설명

### 2.1 문제 정의

사용자 위치를 \(p=(x,y)\)라고 하고, 18개 기지국에서 얻은 RTT 기반 거리 측정값을 \(d_1,d_2,\dots,d_{18}\)이라고 둔다. 강화학습에서는 정답 위치를 직접 출력하지 않고, 에이전트가 현재 위치에서 상하좌우로 이동하거나 멈추는 행동을 반복한다. 에이전트가 멈춘 위치를 최종 예측 위치 \(\hat{p}=(\hat{x},\hat{y})\)로 사용한다.

각 샘플의 위치 오차는 \(e=\sqrt{(x-\hat{x})^2+(y-\hat{y})^2}\)로 계산하였다. 평가 지표는 평균 오차, 중앙값 오차, RMSE, 90% 오차, 최대 오차를 사용하였다. 평균만 보면 일부 샘플에서 크게 실패하는 문제를 놓칠 수 있기 때문에 90% 오차와 최대 오차를 함께 확인하였다.

### 2.2 강화학습 환경 구성

강화학습 환경은 지도학습 데이터셋을 이용하여 구성하였다. 학습 과정에서는 train 데이터에서 사용자 하나를 무작위로 선택하고, 해당 사용자의 실제 위치를 목표 위치로 둔다. 에이전트는 현재 위치와 18개 거리 측정값을 관측하고, 행동을 선택한다. 행동 결과 에이전트의 위치가 바뀌고, 정답 위치와의 거리 변화에 따라 보상이 주어진다.

| 구성 요소 | 설계 내용 |
|---|---|
| 맵 범위 | \(x=-60\sim60\), \(y=-30\sim30\) |
| 상태 | 현재 위치 2개 + RTT 거리값 18개 |
| 상태 차원 | 20차원 |
| 행동 | 위, 아래, 왼쪽, 오른쪽, 멈춤 |
| 행동 개수 | 5개 |
| 이동 크기 | 1.0m |
| 최대 step | 200 |
| 최종 예측 | 에이전트가 STOP을 선택한 위치 |

상태에는 현재 위치 \((x,y)\)와 거리 측정값 18개가 들어간다. 위치값은 맵 범위를 기준으로 정규화하고, 거리값은 일정한 스케일로 나누어 신경망 입력으로 사용하였다. 기지국 위치를 상태에 추가하는 실험도 수행했지만, 평균 성능은 오히려 나빠졌기 때문에 최종 기본 구조에서는 현재 위치와 거리값만 사용하였다.

### 2.3 DQN 모델 구조

행동 공간이 위, 아래, 왼쪽, 오른쪽, 멈춤처럼 이산적이므로 DQN을 사용하였다. DQN은 현재 상태 \(s\)를 입력받아 각 행동 \(a\)에 대한 Q값 \(Q(s,a)\)을 출력한다. 평가 단계에서는 다섯 개 행동 중 Q값이 가장 큰 행동을 선택한다.

본 실험에서 사용한 DQN은 입력층, 두 개의 은닉층, 출력층으로 구성하였다. 은닉층은 각각 128개의 뉴런을 사용하고 활성화 함수는 ReLU를 사용하였다. 출력층의 크기는 행동 개수와 같은 5이다.

DQN의 학습 목표는 현재 Q값이 보상과 다음 상태의 최대 Q값으로 구성된 target에 가까워지도록 만드는 것이다. 기본적인 target은 \(y=r+\gamma\max_{a'}Q_{target}(s',a')\) 형태이다. 여기서 \(r\)은 현재 행동으로 받은 보상, \(\gamma\)는 미래 보상 할인율, \(s'\)는 다음 상태이다. 학습 안정성을 위해 policy network와 target network를 분리하고, 일정 step마다 target network를 업데이트하였다.

| 항목 | 값 |
|---|---:|
| Episodes | 3000 |
| Batch size | 64 |
| Replay memory | 50000 |
| Discount factor \(\gamma\) | 0.98 |
| Learning rate | 0.001 |
| Epsilon start | 1.0 |
| Epsilon end | 0.05 |
| Epsilon decay | 0.995 |
| Target update interval | 100 |

### 2.4 보상 설계

보상 설계는 본 실험에서 가장 중요한 부분이었다. 에이전트는 정답 위치를 직접 알 수 없지만, 학습 중에는 정답 위치를 이용해 행동의 좋고 나쁨을 평가할 수 있다. 이동 행동에 대해서는 이전 위치보다 정답에 가까워지면 양의 보상을 받고, 멀어지면 음의 보상을 받도록 하였다. 또한 불필요하게 오래 이동하는 것을 막기 위해 작은 시간 패널티를 추가하였다.

이동 행동의 보상은 \(R_{move}=\alpha(e_{prev}-e_{curr})-c\)로 정의하였다. 여기서 \(e_{prev}\)는 이동 전 오차, \(e_{curr}\)는 이동 후 오차, \(\alpha\)는 거리 개선 보상의 스케일, \(c\)는 이동 시간 패널티이다. 본 실험의 기본 설정은 \(\alpha=1.0\), \(c=0.01\)이다.

STOP 행동에 대해서는 계단형 보상을 사용하였다. 정답에서 2m 이내에서 멈추면 성공 보상, 5m 이내에서 멈추면 중간 보상, 그보다 먼 곳에서 멈추면 실패 패널티를 부여하였다.

| STOP 위치 조건 | 보상 |
|---|---:|
| 오차 \(\leq 2m\) | +20 |
| 오차 \(\leq 5m\) | +5 |
| 오차 \(> 5m\) | -20 |

부드러운 Gaussian, tanh, sigmoid 계열의 STOP 보상도 실험했지만, 대부분 기본 계단형 보상보다 성능이 나빠졌다. 이는 이 문제에서 DQN이 연속적으로 완만한 보상보다 성공과 실패가 명확하게 구분되는 보상 신호를 더 잘 학습했기 때문으로 해석된다.

## 3. Agent AI 활용 방안

본 프로젝트에서는 Agent AI를 단순히 코드 작성 도구로만 사용하지 않고, 실험 설계와 결과 해석을 반복하는 보조 연구자처럼 활용하였다. 다만 최종 코드 실행, 파라미터 선택, 결과 채택 여부는 직접 판단하였다.

| 단계 | 본인의 역할 | Agent AI의 역할 |
|---|---|---|
| 문제 정의 | RTT 위치 추정 문제를 강화학습으로 바꿔보기로 결정 | 상태, 행동, 보상으로 나누어 설계하는 방향 제안 |
| 환경 설계 | 맵 크기와 입력 데이터 구조 확인 | 맵 범위, 상태 차원, 행동 공간 정리 보조 |
| 알고리즘 선택 | 이산 행동 문제로 구성하기로 결정 | DQN이 적합한 이유 설명 |
| 보상 설계 | STOP 보상, 이동 보상, 실패 패널티를 여러 번 수정 | 각 보상 변경이 왜 성능에 영향을 주는지 해석 보조 |
| 실험 분석 | validation 결과를 직접 실행하고 수치 기록 | 평균, RMSE, 90% error, max error 관점에서 결과 비교 |
| 보고서 작성 | 최종 주장과 실험 선택 기준 결정 | 문장 정리, 표 구성, 한계 분석 정리 보조 |

AI가 특히 도움이 되었던 부분은 실패한 실험을 해석하는 과정이었다. 예를 들어 보상을 부드럽게 만들면 학습이 더 잘될 것이라고 예상했지만, 실제로는 max error가 크게 증가하는 경우가 많았다. 이때 Agent AI를 활용하여 STOP 보상이 너무 완만해져 에이전트가 멈춤 타이밍을 명확히 학습하지 못했을 가능성을 검토하였다.

반대로 AI의 제안을 그대로 따르지 않은 부분도 있다. 예를 들어 기지국 위치를 상태에 추가하면 정보량이 늘어나므로 성능이 좋아질 수 있다는 제안이 있었지만, 실제 실험에서는 평균 오차와 RMSE가 악화되었다. 따라서 최종 보고서에서는 AI 제안 자체가 아니라, 실제 validation 결과를 기준으로 채택 여부를 결정하였다.

## 4. 결과 도출 & 디스커션

### 4.1 비교용 실험 1: 물리 기반 baseline

비교용 실험 1은 RTT 거리값을 먼저 기지국별로 보정한 뒤, Weighted Robust Least Squares로 1차 위치를 추정하고, Ridge Regression으로 residual correction을 수행하는 방식이다. 이 방법은 강화학습과 달리 위치 추정 문제의 기하학적 구조를 직접 사용한다.

물리 기반 baseline에서는 기지국별 평균 오차를 bias로 계산하여 거리값에서 제거하고, 오차 표준편차가 작은 기지국에 더 큰 weight를 부여하였다. 이후 Huber loss를 적용한 robust least squares로 1차 위치를 구하고, 보정된 거리값, 거리값 제곱, WLS 위치, 각 기지국 residual을 feature로 사용하여 Ridge Regression이 잔여 오차를 보정하였다.

최종 비교 기준으로는 validation에서 가장 안정적이었던 ridge alpha 20.0, Huber scale 2.0 결과를 사용하였다.

| 방법 | Mean | Median | RMSE | 90% Error | Max |
|---|---:|---:|---:|---:|---:|
| 물리 기반 baseline | 8.1541 | 7.4217 | 9.3769 | 14.5336 | 25.9557 |

이 결과는 평균 오차와 RMSE 측면에서 DQN보다 좋다. 그 이유는 물리 기반 방법이 기지국 위치와 거리 방정식이라는 문제의 구조를 직접 사용하기 때문이다. 따라서 이 실험은 강화학습 모델의 우수성을 보이기 위한 대상이라기보다, 강화학습 접근이 현재 어느 정도까지 도달했는지 판단하기 위한 비교 기준으로 사용하였다.

전체 결과를 보면 RL 기본 상태 실험이 평균 오차와 RMSE 기준으로 가장 좋았다. 기지국 위치를 상태에 추가하거나, Double DQN을 적용하거나, 부드러운 STOP 보상을 사용하는 시도는 대부분 기대한 만큼의 개선을 만들지 못했다. 특히 부드러운 보상 계열은 max error가 크게 증가하는 경우가 많았다.

### 4.3 최종 선택한 강화학습 설정

최종 DQN 설정은 기본 상태와 계단형 STOP 보상을 사용한 실험이다. 이 설정은 mean error 11.4421m, RMSE 13.3172m, 90% error 19.8977m를 기록하였다.

| 최종 RL 설정 | 값 |
|---|---|
| 알고리즘 | DQN |
| 상태 | 현재 위치 2개 + 거리값 18개 |
| 행동 | 위, 아래, 왼쪽, 오른쪽, 멈춤 |
| 시작 위치 | 학습 시 random, 평가 시 center |
| 이동 보상 | 거리 오차 감소량 기반 |
| STOP 보상 | 2m 이내 +20, 5m 이내 +5, 그 외 -20 |
| 최종 validation mean error | 11.4421m |
| 최종 validation RMSE | 13.3172m |

이 설정을 선택한 이유는 단순히 첫 번째 실험이기 때문이 아니다. 이후 여러 실험에서 보상, 상태, 학습량을 수정했지만 평균 오차와 RMSE가 대부분 악화되었다. 따라서 현재 데이터와 DQN 구조에서는 복잡한 보상보다 명확한 계단형 STOP 보상과 기본 상태 구성이 가장 안정적이었다.

### 4.4 물리 기반 baseline과 DQN 비교

| 방법 | Mean | Median | RMSE | 90% Error | Max |
|---|---:|---:|---:|---:|---:|
| 물리 기반 baseline | 8.1541 | 7.4217 | 9.3769 | 14.5336 | 25.9557 |
| DQN 강화학습 | 11.4421 | 9.9606 | 13.3172 | 19.8977 | 42.7899 |

수치상으로는 물리 기반 baseline이 DQN보다 더 좋은 성능을 보였다. 평균 오차는 약 3.29m, RMSE는 약 3.94m 차이가 났다. 특히 최대 오차도 물리 기반 baseline이 더 낮았다.

하지만 이 비교는 완전히 같은 조건의 알고리즘 비교라고 보기는 어렵다. 물리 기반 baseline은 기지국 위치와 거리 방정식을 직접 사용하고, 기지국별 bias와 weight를 계산하며, Ridge Regression으로 잔여 오차까지 보정한다. 즉 문제의 물리적 구조를 강하게 활용한다. 반면 DQN은 현재 위치와 거리값을 상태로 받아 이동 정책을 학습하는 방식이다. DQN은 직접적인 삼변측량 공식을 사용하지 않고, 이동과 멈춤 행동을 통해 위치를 찾는다.

따라서 본 실험에서 얻은 결론은 “DQN이 물리 기반 방법보다 우수하다”가 아니라, “RTT 위치 추정 문제를 강화학습 문제로 구성할 수는 있지만, 현재 설계에서는 물리 기반 baseline보다 성능이 낮다”이다. 특히 위치 추정처럼 기하학적 구조가 강한 문제에서는 그 구조를 직접 활용하는 방법이 여전히 강력하다.

### 4.5 실패한 실험에서 얻은 해석

첫째, 보상을 약하게 만들면 정책이 애매해졌다. 실험 5에서는 성공 보상과 실패 패널티를 모두 줄였는데, mean error와 max error가 악화되었다. 이는 정답 근처에서 멈추는 행동의 이득과 멀리서 멈추는 행동의 손해가 모두 약해져서 STOP 판단이 흐려졌기 때문으로 보인다.

둘째, 부드러운 STOP 보상은 기대와 다르게 성능을 악화시켰다. Gaussian, tanh, sigmoid 기반 보상은 이론적으로는 더 자연스러운 보상처럼 보이지만, 실제 DQN 학습에서는 계단형 보상보다 명확한 성공/실패 신호를 주지 못했다. 특히 max error가 100m 이상으로 커진 실험도 있었기 때문에, 일부 샘플에서 정책이 완전히 잘못된 행동을 강하게 선택한 것으로 볼 수 있다.

셋째, 상태에 기지국 좌표를 추가하는 것이 항상 좋은 것은 아니었다. 기지국 위치를 추가하면 정보량은 늘어나지만, 상태 차원이 커지고 학습해야 할 관계도 복잡해진다. 실험 14에서는 median은 약간 좋아졌지만 RMSE와 max error가 크게 나빠졌다. learning rate를 낮춘 실험 15에서는 max error가 개선되었지만 mean과 RMSE는 기본 DQN보다 나빴다.

넷째, 학습량을 늘리는 것만으로는 성능이 좋아지지 않았다. episode를 크게 늘리거나 replay memory를 증가시킨 실험에서도 성능이 개선되지 않았다. 이는 현재 문제에서 단순히 더 오래 학습하는 것보다, 상태 표현과 보상 구조 자체가 더 중요하다는 것을 보여준다.

### 4.6 한계 및 future work

본 실험의 가장 큰 한계는 DQN이 위치 추정 문제의 기하학적 구조를 충분히 활용하지 못했다는 점이다. 현재 상태는 현재 위치와 거리값으로 구성되어 있지만, 각 기지국과 현재 위치 사이의 예상 거리, 측정 거리와 예상 거리의 residual, 기지국별 신뢰도 같은 정보는 직접적으로 포함하지 않았다. 이러한 정보는 물리 기반 baseline에서 성능 향상에 중요한 역할을 했기 때문에, 강화학습에서도 상태에 더 구조적으로 포함할 필요가 있다.

또한 현재 행동은 1m 단위의 상하좌우 이동으로 제한되어 있다. 이 방식은 구현이 단순하지만, 실제 위치 추정 문제에서는 연속 좌표를 바로 예측하는 actor-critic 계열 방법이 더 적합할 수 있다. 예를 들어 DDPG, TD3, SAC 같은 연속 제어 알고리즘을 사용하면 \((\Delta x, \Delta y)\) 또는 최종 위치 보정량을 연속값으로 출력할 수 있다.

마지막으로 현재 DQN은 각 샘플을 독립적인 환경으로 보고 학습한다. 그러나 RTT 오차의 분포나 기지국별 신뢰도는 전체 데이터에서 반복적으로 나타나는 구조이므로, 강화학습과 지도학습을 결합한 hybrid 방식도 고려할 수 있다. 예를 들어 물리 기반 WLS 결과를 초기 위치로 사용하고, DQN은 그 주변에서 보정 행동만 수행하게 만들면 탐색 공간이 줄어들어 성능이 개선될 가능성이 있다.
